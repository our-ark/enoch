# Enoch v0.1.0

Enoch v0.1.0 is the first public reference software body in the Our-Ark agent
lineage.

## Included

- A persistent, identity-bearing local agent body.
- Core `code`, `work`, `evolve`, `learn`, `inherit`, and `skill-library`
  capabilities, with provider-owned reference integration skills.
- Six evolution candidate sources: backlog, feedback, experience, inheritance,
  learning from other agents, and theme-based brainstorming.
- Human-reviewed candidate, task, promotion, and adoption lifecycle support.
- Isolated task worktrees, bounded retries, provenance events, and hermetic
  end-to-end evolution tests.
- Provider abstractions for chat, agent runtime, version control, code forge,
  and host-service integrations, with load-time contract validation.
- Replaceable provider-owned runtime configuration and runtime-neutral task
  pause and resume behavior.
- A network-free wheel installation E2E using independent third-party chat and
  VCS provider distributions.
- A Genesis schema version 1 body manifest with pinned, replaceable shared
  runtime dependencies.
- Fresh private-state boundaries for Genesis-created descendants.

## Compatibility

- Python 3.11, 3.12, or 3.13 tested (`requires-python >=3.11`).
- `genesis.toml` schema version 1.
- Genesis v0.1.0 is the matching creation-engine release.

Credentials, memories, logs, chat identifiers, and instance configuration are
not part of the public or inheritable software body.
